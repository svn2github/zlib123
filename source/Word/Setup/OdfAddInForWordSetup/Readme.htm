OpenXML/ODF Translator Add-in for Microsoft Word

Thank you for downloading the OpenXML/ODF Translator Add-in for Word. The add-in enables you to open documents based on the ODF specification in Microsoft Word, thus, demonstrating the interoperability between Office Open XML & ODF. This interoperability project is hosted on sourceforge.net. For more details and future updates on the project, please see the RoadMap document.

Funded by Microsoft the OpenXML/ODF Translator has been developed by Clever Age, Aztecsoft, Sonata and DIaLOGIKa (for more information, see the project web site).

Software Requirements

Before installing the add-in, make sure you have one of the following ...

  • Microsoft Word 2000
  • Office Compatibility Pack
  • .NET Framework 2.0

or

  • Microsoft Word XP
  • Office Compatibility Pack
  • .NET Framework 2.0

or

  • Microsoft Word 2003
  • Office Compatibility Pack
  • .NET Framework 2.0*

or

  • Microsoft Word 2007 (with .NET Programmability Support activated)
  • .NET Framework 2.0*

(*) Note: if you installed Microsoft Word before .NET Framework, you will need to update your Word installation by doing the following:

  1. Open Add or Remove Programs from the Control Panel
  2. Find the Microsoft Word entry and choose Modify
  3. In the installation process, choose Add or remove features then add .NET Programmability Support in the Microsoft Office Word section.

Installing the Add-in

Double click the "OdfAddInForWordSetup-en.exe" file

If the installation is successful, you should be able to open an .ODT file (ODT is the standard file extension for ODF text documents) using Microsoft Word (see File>ODF... menu). Also, when right-clicking on an .ODT file in Explorer, you should have the option of opening the document in Microsoft Word (Open with Word menu).

Important note: The .ODT file opened by the add-in is imported in Microsoft Word as a read-only .DOCX file. Consequently, if you have made changes to it and want to save it back as an .ODT file, you will have first to save it with a new name (File>Save As) and then use the File>ODF…-Save as ODF… command.

Troubleshooting

If you don't see the ODF entry in the Microsoft Word File menu, this might be due to missing features in Word (see Software Requirements section) or a failure during the installation of the Word Primary Interop Assemblies.

Please check our troubleshooting guide for more information.

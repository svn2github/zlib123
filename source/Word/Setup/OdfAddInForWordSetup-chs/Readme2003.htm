ODF Add-in for Microsoft Word

感谢您下载 ODF Add-in for Word。 该软件的目的在于使用户能够用 Microsoft Word 打开基于 ODF 规格的文档，并展示 OpenXML 和 ODF 规格之间的互操作性。 互操作性项目托管在sourceforge.net上。 如需有关此项目的更多细节和未来更新，请查看“路线图（RoadMap）”文档。

此加载项由Microsoft出资，由Clever Age，DIaLOGIKa 和 Aztecsoft 开发（请参见项目网站，以获取更多信息）。

软件要求

在安装该加载荐之前，确保您拥有下列项目之一...

  • Microsoft Word 2000
  • Office 兼容包
  • .NET framework 2.0

或者

  • Microsoft Word XP
  • Office 兼容包
  • .NET framework 2.0

或者

  • Microsoft Word 2003
  • Office 兼容包
  • .NET framework 2.0*

或者

  • 具有已激活 .NET 可编程性支持的Word 2007
  • .NET framework 2.0*

（*）警告： 如果您先安装 Microsoft Word，再安装 .NET Framework，您将需要通过运行以下步骤，更新您的 Word 安装。

  1. 从控制面板打开“添加或删除程序”
  2. 找到 Microsoft Word 条目并选择“修改”
  3. 在安装进程中，选择“添加或删功能”，然后在“Microsoft Office Word”部分中添加“.NET 可编程支持”。

安装此加载项

双击“OdfAddInForWordSetup-en-1.0.exe” 文件

如果安装成功，您应该能够从 Microsoft Word 中打开“.ODT”文件（参见菜单“FileODF...”）。 同时，当在资源管理器中右击“.ODT”文件时，您应该有在 Microsoft Word 中打开该文档的选项（菜单“用 Word 打开”）。

重要注释： 由该加载项打开的 ODF 文件，可在 Microsoft Word 中作为只读 .DOCX 文件导入。 如果您想将其保存为 ODF，您将必须首先为该文档创建一个副本（“菜单另存为”），然后使用“导出成 ODF”菜单。

故障排除

如果您未在 Microsoft Word 菜单中看到 ODF 项，这可归因于 Word 缺少功能（参见 软件要求 部分）或者未能安装 Word 主互操作程序集。 那么您将需要通过执行以下步骤，手动激活加载项...

Microsoft Word 2000/XP/2003

  1. 转到工具 gt; 自定义，然后单击“命令”选项卡。
  2. 在“类别”框中，单击“工具”。
  3. 将 COM 加载项从“命令”框拖至“工具”菜单。 当“工具”菜单显示菜单命令时，指向您希望显示 COM 加载项命令的菜单位置，然后松开鼠标按钮。
  4. 单击“关闭”。
  5. 在“工具”菜单中，单击 COM 加载项，并选定可用列表加载项名称旁的复选框。 如果您想要的加载项未在列表上，单击“添加”，找到该加载项，然后单击“确定”。

Microsoft Word 2007

  1. 在“文件”菜单中单击“Word 选项”
  2. 在“加载项”部分中，选择列表框中的“COM 加载项”，并按“转到”按钮
  3. 确保选择复选框“ODF Word 加载项”，并用“确定”按钮验证。
